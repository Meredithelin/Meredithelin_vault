# 飞机大战 - 星际守护者 (Star Guardian)

> **🚀 立即体验: [https://flywar.netlify.app/](https://flywar.netlify.app/)**  
> 欢迎所有指挥官加入战斗，挑战极限，守护星际和平！

## 🎮 游戏简介
《飞机大战 - 星际守护者》是一款基于 HTML5 Canvas 开发的快节奏横版射击游戏。玩家将驾驶高科技战机，在赛博朋克风格的星际战场中对抗无穷无尽的敌人，挑战强大的 Boss，并守护宇宙的和平。

游戏支持单人模式和本地双人合作模式，拥有丰富的战机升级系统、技能系统和成就系统。

## ✨ 核心特色

### 1. 多样化的战机系统
游戏提供 4 种各具特色的战机供玩家解锁和升级：
- **苍穹之翼 (Default)**: 均衡型战机，适合新手，拥有稳定的弹道。
- **红莲重炮 (Blast)**: 力量型战机，移动较慢但火力惊人，发射高爆火球。
- **闪电战机 (Lightning)**: 速度型战机，拥有极高的机动性和高射速的等离子尖刺。
- **守护战机 (Guardian)**: 防御型战机，拥有高生命值，容错率极高。

### 2. 深度升级与经济系统
- **金币系统**: 击杀敌人可获得金币，用于购买新战机或强化现有战机。
- **等级强化**: 每种战机最高可升至 Lv.5，升级后不仅属性提升，外观和子弹特效也会进化。

### 3. 激爽战斗体验
- **技能系统**:
  - **全屏炸弹 (Bomb)**: 瞬间清除屏幕上所有敌机和子弹，并对 Boss 造成重创。
  - **无敌金身 (Invincible)**: 开启后获得 5 秒绝对无敌时间，并附带金色光环特效。
- **狂暴模式**: 拾取特定道具可进入狂暴状态，火力全开，弹幕覆盖全屏。
- **BOSS 战**: 关卡末尾将遭遇拥有多阶段形态的强大 Boss，考验玩家的操作与反应。

### 4. 双人合作 (Local Co-op)
支持本地双人同屏作战，P1 和 P2 可以选择不同的战机互补，共同挑战更高难度。

## 🕹️ 操作指南

| 功能 | 玩家 1 (P1) | 玩家 2 (P2) |
| :--- | :--- | :--- |
| **移动** | W, A, S, D | ↑, ↓, ←, → |
| **射击** | 空格键 (Space) | 回车键 (Enter) |
| **全屏炸弹** | Q | , (逗号) |
| **无敌金身** | E | . (句号) |

*   **暂停游戏**: 按 `P` 键或点击屏幕左下角的暂停按钮。
*   **投降**: 点击界面上的投降按钮可快速结束当前局（结算分数）。

## 📦 道具系统

在战斗中，击毁敌机有几率掉落强力补给：
- **🟢 武器升级**: 提升子弹等级（最高 3 级），满级后获得狂暴效果。
- **🔴 生命恢复**: 恢复 1 点生命值。
- **🔵 能量护盾**: 获得一层抵挡伤害的护盾。
- **🟠 狂暴模式**: 短时间内射速与弹幕大幅增强。
- **⚫ 炸弹补给**: 增加 1 次炸弹使用次数。
- **🟡 金身补给**: 增加 1 次金身使用次数。

## 👾 敌人图鉴
- **普通敌机 (红)**: 基础单位，直线飞行。
- **突击敌机 (黄)**: 速度极快，体型小巧。
- **重装敌机 (紫)**: 血量厚，移动慢，提供大量分数。
- **射手敌机 (青)**: 会发射追踪或散射子弹，极具威胁。
- **最终 BOSS**: 拥有巨型机身和多重攻击阶段（弹幕、激光、召唤）。

## 🛠️ 技术栈
- **核心**: 原生 JavaScript (ES6+)
- **渲染**: HTML5 Canvas API
- **样式**: CSS3 (Flexbox, Grid, Animations)
- **存储**: LocalStorage (用于保存金币、解锁进度、成就和最高分)

## 🚀 如何开始
1. 直接用浏览器打开 `index.html` 文件。
2. 在主菜单输入指挥官代号（用于记录成就）。
3. 点击“开始游戏”进入战机选择界面。
4. 选择心仪的战机，点击“出击”！

## 10. 技术难点及解决方案

### 10.1 性能优化
- **问题**：大量敌人和子弹同时存在时性能下降
- **解决方案**：使用对象池管理游戏对象，减少内存分配和垃圾回收

### 10.2 碰撞检测
- **问题**：复杂形状的碰撞检测计算量大
- **解决方案**：使用层次化碰撞检测，先进行粗略检测，再进行精确检测

### 10.3 敌人AI
- **问题**：BOSS的复杂行为模式实现困难
- **解决方案**：使用状态机管理BOSS行为，每个状态对应不同的攻击模式

# Gemini CLI 综合使用教程

欢迎使用 Gemini CLI！这是一个强大的、基于 AI 的命令行界面（CLI）助手，旨在通过自然语言指令，帮助您更高效地完成编码、文件管理、代码库理解和日常开发任务。

本教程将由易到难，带您逐步了解 Gemini CLI 的核心功能和使用技巧。

## 基础篇：与 Gemini 对话

这是最基本也是最核心的用法。您可以像和聊天机器人对话一样，向 Gemini 提出问题、寻求解释或获取信息。

### 1. 启动交互式会话
直接在您的终端中输入：
```bash
gemini
```
这将启动一个对话会话。您可以开始提出任何问题，例如：
> "解释一下什么是 RESTful API？"
> "帮我写一个 Python 函数，用来检查一个单词是否是回文。"

### 2. 快速、一次性的问题
如果您只想快速得到一个答案而不想进入完整的对话模式，可以直接将问题作为参数传递：
```bash
gemini "将 'Hello World' 翻译成西班牙语"
```

### 3. 理解网页内容
您可以让 Gemini 读取并总结一个网页。
```bash
gemini "总结一下这个页面: https://example.com"
```

## 进阶篇：与您的文件系统和代码库交互

Gemini 的强大之处在于它能够理解您的项目上下文，并直接与文件系统进行交互。您可以通过自然语言让它完成文件查找、读取和修改等操作。

### 1. 浏览和列出文件
您可以让 Gemini 帮您寻找文件，它会智能地使用 `glob` 或 `list_directory` 等工具。
> "列出当前目录下所有的 Markdown 文件。"
> "在 `src` 文件夹下查找所有以 `service.ts` 结尾的文件。"

### 2. 读取文件内容
想知道一个文件里写了什么？直接告诉 Gemini。
> "帮我看看 `package.json` 这个文件里有什么内容。"
> "阅读 `src/utils.js` 并告诉我 `calculatePrice` 函数是如何实现的。"

### 3. 在代码库中搜索
这是最常用的功能之一。当您想在整个项目中查找某个函数、变量或特定文本时：
> "在整个项目中搜索 `API_ENDPOINT` 这个变量在哪里被定义了。"
> "查找所有调用了 `getUserProfile` 函数的地方。"

### 4. 修改文件内容
Gemini 可以直接修改文件。为了确保修改的精确性，**最好提供清晰的上下文**。

**示例：**
> "在 `config.js` 文件中，把 `const DEBUG = true` 修改为 `const DEBUG = false`。"

Gemini 会找到对应的代码行，并使用新代码替换旧代码。对于复杂的修改，您可以先让它读取文件，然后给出修改指令。

## 高级篇：执行命令与自动化工作流

Gemini 能够执行 shell 命令，并能将多个步骤组合成一个完整的工作流，极大地提升了开发效率。

### 1. 执行 Shell 命令
您可以让 Gemini 帮您运行任何终端命令。为了安全起见，在执行任何可能修改系统状态的命令（如 `npm install`, `git commit`）之前，**Gemini 会向您解释该命令的作用并请求您的确认**。

> "帮我安装一个新的 npm 包 `axios`。"
> (Gemini 会提议运行 `npm install axios`)

> "启动本地开发服务器。"
> (Gemini 会根据项目类型，可能会提议运行 `npm run dev` 或 `python manage.py runserver`)

### 2. Git 版本控制工作流
Gemini 可以帮您处理 Git 操作。

**示例：提交一次修改**
1.  您："刚才的修改都完成了，帮我提交吧。"
2.  Gemini 会运行 `git status` 查看文件状态。
3.  Gemini 可能会问您需要提交哪些文件，然后运行 `git add ...`。
4.  Gemini 会为您草拟一条符合项目风格的提交信息（Commit Message）。
5.  在您确认后，Gemini 才会运行 `git commit` 命令。

### 3. 修复 Bug 的工作流
这是 Gemini 发挥巨大作用的场景。

**示例：修复一个失败的测试**
1.  您："运行单元测试。"
2.  Gemini 运行测试命令（如 `npm test`），并发现一个测试失败了。
3.  您："这个测试失败了，帮我看看是什么原因。"
4.  Gemini 会阅读测试的错误输出，定位到相关的代码文件和行号。
5.  Gemini 分析问题代码，并向您提出修改建议。
6.  您："同意这个修改。"
7.  Gemini 应用修改后，再次运行测试，直到测试通过。

## 核心概念

### 工具 (Tools)
Gemini 的所有能力（读写文件、执行命令、搜索）都由其内部的 "工具" 提供。您通过自然语言下达的指令，最终会被 Gemini 翻译成一个或多个工具的调用。

### 上下文 (Context)
Gemini 了解您当前的工作目录和项目结构。这使得它在回答问题或执行操作时，能够结合具体的项目环境，给出更精准的答案。

### 安全性
您的安全是第一位的。对于所有可能产生副作用的命令（特别是文件修改和 shell 命令），Gemini 都会在执行前进行提示，并需要得到您的明确批准。
